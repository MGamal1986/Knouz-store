Add complete bilingual support (Arabic and English) to the existing كنوز (Knouz) e-commerce project. The website must support both languages with a toggle switcher. Do not break any existing functionality.

---

I18N LIBRARY

Use next-intl for internationalization:
- Run: npm install next-intl
- Supported locales: ar (default), en
- Default locale: ar
- URL structure: /ar/... and /en/...
- Root / redirects to /ar automatically

---

FILE STRUCTURE

Create this translation file structure:

messages/
├── ar.json
└── en.json

Create next.config.js with next-intl plugin:
import createNextIntlPlugin from 'next-intl/plugin'
const withNextIntl = createNextIntlPlugin()
export default withNextIntl({ ... existing config ... })

Create i18n.ts at root:
import { getRequestConfig } from 'next-intl/server'
export default getRequestConfig(async ({ locale }) => ({
  messages: (await import(`./messages/${locale}.json`)).default
}))

Update middleware.ts to handle locale routing alongside existing admin protection:
- Use createMiddleware from next-intl/middleware
- Locales: ['ar', 'en']
- defaultLocale: 'ar'
- Preserve admin role protection for /[locale]/admin routes

Move all pages inside app/[locale]/ folder:
app/
└── [locale]/
    ├── layout.tsx        ← set dir and lang from locale
    ├── page.tsx
    ├── shop/page.tsx
    ├── product/[slug]/page.tsx
    ├── cart/page.tsx
    ├── checkout/page.tsx
    ├── track/page.tsx
    ├── login/page.tsx
    ├── register/page.tsx
    ├── account/page.tsx
    └── admin/
        ├── layout.tsx
        ├── page.tsx
        ├── products/page.tsx
        ├── orders/page.tsx
        ├── customers/page.tsx
        ├── coupons/page.tsx
        ├── categories/page.tsx
        └── settings/page.tsx

In app/[locale]/layout.tsx:
- Read locale param
- Set <html lang={locale} dir={locale === 'ar' ? 'rtl' : 'ltr'}>
- Wrap with NextIntlClientProvider passing messages

USING TRANSLATIONS IN COMPONENTS

In every page and component, replace all hardcoded Arabic strings with translation calls:

import { useTranslations } from 'next-intl'

export default function Header() {
  const t = useTranslations('nav')
  return (
    <nav>
      <a href="/">{t('home')}</a>
      <a href="/shop">{t('shop')}</a>
    </nav>
  )
}

For server components use getTranslations:
import { getTranslations } from 'next-intl/server'

export default async function Page() {
  const t = await getTranslations('home')
  return <h1>{t('hero_title')}</h1>
}

NUMBER AND CURRENCY FORMATTING

Create lib/formatters.ts:

export function formatPrice(amount: number, locale: string): string {
  if (locale === 'ar') {
    return `${amount.toLocaleString('ar-EG')} جنيه`
  }
  return `EGP ${amount.toLocaleString('en-US')}`
}

export function formatDate(date: Date, locale: string): string {
  return date.toLocaleDateString(locale === 'ar' ? 'ar-EG' : 'en-US', {
    year: 'numeric',
    month: 'long',
    day: 'numeric'
  })
}

Use these helpers everywhere prices and dates are displayed.
